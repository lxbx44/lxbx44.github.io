Flyser: Easily Analyze and Extract Insights from Your Files

Welcome to Flyser! A simple file analyzer developed in Rust. With Flyser, you can easily analyze various file properties and get valuable information about your files. Let's explore how you can install and use Flyser to analyze your files.

Installation

To get started with Flyser, follow the installation instructions below:

Requirements

  • Rust
  • Cargo

Install Rust and Cargo

Before installing Flyser, make sure you have Rust and Cargo installed on your system. Here's how to install them:

Linux / macOS:

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

source $HOME/.cargo/env

Windows:

Download the Rust installer from the official website. Follow the installation instructions to install Rust and Cargo.

Clone and Build

  1. Clone the Flyser repository:

git clone https://github.com/lxbx44/flyser.git

cd flyser/

  1. Build the project:

cargo build --release

Usage

To start using Flyser, run the following command in the project directory:

flyser [file name]

Replace [file name] with the name or path to the file you want to analyze. For example, if you want to analyze the file "src/main.rs", use the following command:

flyser src/main.rs

Once you run the command, Flyser will analyze the file and display information such as file name, file type, file path, file size, number of lines, number of words, and number of characters in a formatted table.

How Does the Program Work?

Flyser leverages the Rust programming language to provide file analysis capabilities. Let's take a closer look at some key aspects of the program:

Parsing Command-Line Arguments

Flyser utilizes the `clap` crate to parse command-line arguments. Here's an example of the `Args` structure used for argument parsing:

use clap::Parser;

#[derive(Parser, Debug)]
#[command(author, version, about, long_about = None)]
struct Args {
    file_name: String,
}

Opening and Analyzing Files

Once the command-line arguments are parsed, Flyser opens and analyzes the specified file. It extracts various properties, such as file type, file size, number of lines, words, and characters.

Here's an example of opening and analyzing a file in Flyser:

use std::fs::File;
use std::io::BufRead;
use std::io::BufReader;

// ...

let file = match File::open(&file_path) {
    Ok(file) => file,
    Err(_) => {
        println!("Could not open the file!");
        return;
    }
};

let file_buffer = BufReader::new(&file);

let (line_count, word_count, char_count) =
    file_buffer.lines().fold((0, 0, 0), |counts, line_result| {
        let line = line_result.unwrap_or_else(|_| String::new());
        (
            counts.0 + 1,
            counts.1 + line.split_whitespace().count(),
            counts.2 + line.chars().count(),
        )
    });

// Display the analyzed file properties
// ...

Formatting and Displaying Results

Once the file properties are extracted, Flyser formats the results and displays them in a table. It utilizes the `comfy_table` crate for generating formatted tables.

Conclusion

Flyser offers a simple and efficient way to analyze your files. By using this tool, you can quickly obtain important file information and gain insights into your data. Give Flyser a try and simplify your file analysis process!
